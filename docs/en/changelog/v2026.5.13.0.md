# GensokyoAI v2026.5.13.0 Changelog

Release date: 2026-05-13

## One-Sentence Summary

This is the first public Alpha release of GensokyoAI. It is not an incremental update over a previous version, but establishes the initial baseline of externally integrable capabilities: roleplay Agent, provider abstraction, sessions and memory, tool calling, character packages, Runtime RPC, HTTP / WebSocket entry points, config and character diagnostics, migration diagnostics, and resource control.

## New Features

- First release of the GensokyoAI Python backend toolkit and Runtime API.
  - User impact: you can run the roleplay Agent from source or package entry points, and integrate via Runtime RPC into CLI, desktop clients, web services, or third-party programs.
  - Action needed: Python 3.14+ is required; we recommend using uv to manage the environment.

- Multi-provider model access capability.
  - Supports Ollama, OpenAI, OpenRouter, DeepSeek, OpenAI Responses, Claude, and Gemini.
  - Provider SDKs remain optional dependencies; the minimal install does not force installation of all model service dependencies.
  - Local Ollama is recommended by default and can be installed via the `default` or `ollama` extra.

- Core roleplay Agent capabilities.
  - Supports character YAML, persona prompts, greetings, example dialogues, session context, tool calling, and streaming replies.
  - Supports `agent.init`, `agent.send_message`, and `agent.send_message_stream` Runtime RPC.

- Session, memory, and export capabilities.
  - Supports creating, listing, resuming, querying current, deleting, exporting, renaming, and rolling back sessions.
  - Supports semantic memory list, search, read, update, delete, and topic graph queries.
  - Session export includes machine-readable schema information for subsequent client and toolchain processing.

- Config, character, and character package diagnostics.
  - `config.validate` returns structured configuration diagnostics before initializing the Agent.
  - `character.validate` validates and previews character YAML.
  - `character.list` lists characters and retains diagnostics for broken files.
  - `.gensokyo-character` character packages support validate, preview, import, and export.

- Runtime integration entry points.
  - JSON Lines RPC: `bridge_main.py`.
  - HTTP / WebSocket adapter: `runtime_http.py`.
  - `GET /info` / `runtime.info` exposes package version, protocol version, methods, legacy methods, method specs, capabilities, schema versions, migration diagnostics, and resource control summary.
  - WebSocket `agent.send_message_stream` can push Runtime stream results event by event.

- Resource control and structured errors.
  - Runtime entry points and deep model / tool calls have resource gates and timeout configuration.
  - When resource limits are triggered, stable `resource.limit_exceeded` structured errors are returned.
  - Tool errors, Runtime RPC errors, and method-not-found errors all provide stable error codes, user prompts, recovery suggestions, and details.

## Behavior Changes

- First release, no behavior changes relative to a previous official version.
- The current version is in Alpha: core capabilities are ready for integration validation, but documentation, client compatibility, and end-to-end acceptance may continue to be tightened before a stable release.
- Runtime RPC recommends namespaced methods; old non-namespaced methods remain compatible but are marked deprecated.

## Bug Fixes

- First release, no fixes relative to a previous official version.
- Pre-release tightening corrected documentation descriptions of Runtime streaming capabilities that were too future-oriented or inaccurate.
- Pre-release tightening adjusted abstract base class placeholder implementations from simple `pass` to explicit `raise NotImplementedError`, and added abstract contract tests.

## Deprecated but Still Compatible

- Legacy non-namespaced Runtime RPC methods are deprecated but still compatible.
  - Deprecated objects: `init`, `send_message`, `send_message_stream`, `list_characters`, `create_session`, `list_sessions`, `current_session`, `resume_session`, `delete_session`, `export_session`, `rename_session`, `rollback_session`, `shutdown`, `dependency_status`, `install_dependencies`, `external_tool_status`.
  - Effective version: `2026.5.13.0`.
  - Alternative: use the namespaced method pointed to by `runtime.info.method_specs[].replacement`, e.g. use `agent.init` instead of `init`, `agent.send_message` instead of `send_message`.
  - Planned removal: `2.0.0`.
  - Runtime declaration: exposed through `runtime.info.deprecated_methods` and `runtime.info.compatibility_notes`.

## Removed or Breaking Changes

- None.
  - Runtime protocol major remains `1`.
  - `runtime.info.breaking_changes` is currently an empty array.

## Data Migration and Upgrade Notes

- First release does not include an upgrade path from an old official version to a new official version.
- Current implementation supports migrating old unversioned sessions or old memory topic stores to the current schema when read, and records migration diagnostics.
- Migration diagnostics can be viewed via `runtime.info.migration_diagnostics`.
- It is recommended to back up `config/local.yaml`, `characters/`, `sessions/`, memory data, and custom resources before upgrading or switching versions.

## Installation and Dependency Changes

- Python version requirement: Python 3.14+.
- Recommended package manager: uv.
- Minimum runtime dependencies are in `requirements.txt` and `pyproject.toml` `dependencies`.
- Provider SDKs are optional dependencies:
  - `default` / `ollama`: Ollama.
  - `openai`, `openrouter`, `deepseek`, `openai_responses`: OpenAI SDK.
  - `claude`: Anthropic SDK.
  - `gemini`: Google GenAI SDK.
  - `all`: installs all built-in Provider SDKs.
- Windows users can use `run_default_uv.cmd` or `run_default_pip.cmd`. Before using `run_default_pip.cmd`, install the corresponding provider extra, e.g. `python -m pip install -e .[default]`.

## Runtime / Client Compatibility

- Runtime protocol version: `1.0.0`.
- Runtime protocol major: `1`.
- Supported clients: it is recommended that clients call `runtime.info` for capability discovery at startup.
- Method changes: first-release baseline includes the following non-legacy methods:
  - `runtime.info`, `runtime.health`, `runtime.shutdown`
  - `config.validate`
  - `character.validate`, `character.list`
  - `character_package.validate`, `character_package.preview`, `character_package.import`, `character_package.export`
  - `agent.init`, `agent.send_message`, `agent.send_message_stream`
  - `model.list`, `model.info`
  - `session.create`, `session.list`, `session.current`, `session.resume`, `session.delete`, `session.export`, `session.rename`, `session.rollback`
  - `dependency.status`, `dependency.install`
  - `external_tool.status`
  - `memory.list`, `memory.search`, `memory.get`, `memory.update`, `memory.delete`, `memory.graph`
- Capability changes: first-release baseline includes agent lifecycle / messaging / streaming, character discovery / validation, character package management, dependency management, external tool status, memory management / search / graph, model discovery, config validation, migration diagnostics, resource control, runtime events / health / versioning, session management.
- Response field changes: first-release baseline follows `docs/en/runtime_api.md`.
- Deprecated methods / fields: legacy non-namespaced RPC methods are deprecated; `deprecated_fields` is currently an empty array.
- Compatibility notes: legacy methods are still available, but new clients should migrate to namespaced methods.
- Breaking changes: none.
