# GensokyoAI v2026.7.14.0 Changelog

Release date: 2026-07-14

## Release scope

This is the cumulative official release since the only previous official release, `v2026.5.13.0`. Documents for `v2026.6.21.0` through `v2026.6.25.1` and `v2026.7.4.0` are unpublished development snapshots or candidate notes, not separate official releases; this document consolidates the changes that are present in the current code baseline.

## Summary

This cumulative release adds initiative timers, session-history editing and regeneration, multimodal messages, extended character metadata, default DuckDuckGo search, and a complete scene system. It also completes the Runtime HTTP/WebSocket entry migration with compatibility, security and performance hardening, the license change, and net removal of Streamlit.

## Features and behavior changes

- **Initiative timers**: after a reply, the AI can schedule a later proactive message, with pending summaries, update, cancellation, immediate triggering, hesitation reconsideration, natural fallback, and CLI/Runtime state management.
- **Session-history editing and regeneration**: clients can read messages, replace the complete history, and regenerate from a selected position to implement safe history editing and retry flows.
- **Multimodal support**: message and provider conversion paths accept multimodal content while respecting individual provider capability boundaries.
- **Expanded character data**: character cards and package flows support richer `metadata` and `example_dialogue`; validation and preview APIs expose corresponding summaries.
- **HTTP/WebSocket migration compatibility entry points**: the web implementation moved to `GensokyoAI.backends.web_server`; existing launch scripts and compatibility entry points remain available, with HTTP RPC, WebSocket streaming, and event subscriptions sharing the Runtime contract.
- **Default search**: built-in web search now defaults to DuckDuckGo/DDGS and requires no search-service secret.
- **Scene system**: adds YAML scene libraries, `SceneManager`, persisted current scene per session, scene-switch events, optional connectivity constraints, built-in model tools, and structured `begin_scene`.
- **Scene RPC**: adds `scene.current`, `scene.list`, `scene.get`, `scene.switch`, and `scene.graph` for frontend querying, orchestration, and visualization.
- **Security hardening**: strengthens Runtime Origin/token deployment validation, command and path policies, external-tool policy, prompt-injection boundaries, session persistence paths, and SSRF/URL validation. Async URL validation avoids blocking the HTTP Runtime.
- **Performance work**: reuses HTTP connections, adopts high-performance serialization and async file I/O, caches character configuration, reduces lock scope, and improves concurrent state handling. No percentages are claimed because they were not re-benchmarked during this release process.
- **License change**: the project license changed to Apache License 2.0, aligned across distribution metadata and the repository license file.
- **Net Streamlit removal**: the Streamlit frontend that appeared during development and all remaining integration residue were removed from the final cumulative release; Streamlit is not shipped as a feature or dependency.

## Runtime and client compatibility

- Python package: `2026.7.14.0`; planned corresponding tag: `v2026.7.14.0`.
- Runtime protocol: `1.1.0`; `protocol_major_version` remains `1`.
- **14 non-legacy RPC methods** are new since the `v2026.5.13.0` baseline:
  - Session: `session.messages`, `session.replace_messages`, `session.regenerate_from` (3).
  - Initiative timer: `initiative_timer.current`, `initiative_timer.update`, `initiative_timer.cancel`, `initiative_timer.trigger`, `initiative_timer.hesitation`, `initiative_timer.hesitation.set` (6).
  - Scene: `scene.current`, `scene.list`, `scene.get`, `scene.switch`, `scene.graph` (5).
- Existing namespaced RPC and legacy compatibility methods were not removed. `runtime.info.breaking_changes` remains empty; there is no RPC breaking change.
- Clients should call `runtime.info` first and negotiate against returned `methods`, `method_specs`, `capabilities`, and schema versions.

## Data migration and schemas

- Memory schema advances from official baseline `1` to `2`. Reading an old memory topic store automatically migrates it to schema 2 and creates a pre-migration backup; success or failure is reported through `runtime.info.migration_diagnostics`.
- If migration fails, the original error is reported and the diagnostic contains stable details. Keep the source file and `.bak` backup, then repair or roll back based on diagnostics instead of deleting the backup.
- All other public schemas remain at `1`: config, session, session export, and character package.
- Back up configuration, characters, sessions, memory, and custom scene resources before upgrading.

## Installation, dependencies, and deployment reminders

- The minimum `ayafileio` version is consistently `>=1.4.6`.
- Core dependencies add `ddgs>=9.14.4` for default DuckDuckGo search.
- Provider SDKs remain optional dependencies; minimal installation does not force every provider SDK.
- For browser or cross-origin Runtime clients, explicitly configure allowed Origins and do not permit untrusted sites through an overly broad policy.
- Enable and securely inject an access token whenever HTTP/WebSocket Runtime is exposed beyond localhost or to an untrusted network. Never place tokens in the repository, logs, screenshots, or public frontend assets.
- PyPI upload is deferred for this stage; this preparation covers code, documentation, tests, and local artifacts only.

## Removed or breaking changes

- The Streamlit frontend was removed completely.
- Runtime RPC has no breaking change; protocol major remains `1` and breaking changes remain empty.
