# GensokyoAI v2026.6.21.0 更新日志

发布日期：2026-06-21

## 一句话总结

HTTP / WebSocket Runtime 入口迁移到 `GensokyoAI/backends/web_server`，内置联网搜索工具默认改用 DuckDuckGo（`ddgs`），并优化了主动定时器触发时的系统提示词，让小模型更不容易把“到点主动开口”误判为“用户在重复提问”。

## 新增功能

- 新增 DuckDuckGo 搜索 Provider。
  - 对用户的影响：开启 `tool.web_search` 后，默认不再需要 Bing HTML 解析，也无需 API Key，直接通过 `ddgs` 包调用 DuckDuckGo 搜索。
  - 是否需要操作：如果已安装项目依赖，`ddgs>=9.14.4` 会自动随项目安装；否则重新执行 `uv sync` 或 `python -m pip install -e .[default]` 即可。

- `tool.web_search.provider` 新增可选值 `ddg`，并作为默认值。
  - `ddg`：DuckDuckGo 搜索（同步 API 在异步 Runtime 中通过 `asyncio.to_thread` 包装）。
  - `api`：通用 JSON API 搜索（Tavily、BoCha 等）。
  - `mixed`：并行执行 `ddg` 与 `api`，排序、去重、截断后返回。
  - `bing`：原 Bing HTML 搜索，作为兼容保留。

- 主动定时器触发时新增“无新用户输入”语义提示。
  - 对用户的影响：使用较小模型或廉价模型时，主动回话更自然，出现“不要重复问”“又问一遍？”等出戏回复的概率降低。
  - 是否需要操作：无需改配置，已随代码更新生效。

## 行为变化

- HTTP / WebSocket Runtime adapter 实现迁移到 `GensokyoAI/backends/web_server`。
  - 对用户的影响：推荐启动命令从 `python runtime_http.py` 改为 `python -m GensokyoAI.backends.web_server`。
  - 兼容性：`runtime_http.py` 仍作为兼容包装器保留，旧命令继续可用；`run_runtime.cmd` 也已改为新启动方式。
  - 客户端影响：端点、协议、方法列表均无变化。

- 内置 `web_search` 工具默认 Provider 由 `bing` 改为 `ddg`。
  - 对用户的影响：开启 `tool.web_search.enabled` 后，默认走 DuckDuckGo；如需继续使用 Bing 或 API，请显式设置 `provider: "bing"` 或 `provider: "api"`。
  - 兼容性：`bing`、`api`、`mixed` 仍可用；`mixed` 现在合并 `ddg` 与 `api` 结果。
  - 客户端影响：工具 schema、返回结构、错误码不变；`diagnostics` 中 `provider` 字段会显示 `ddg`。

- 主动回话提示词明确区分“无新用户输入”与“继续自己的上一句话”。
  - 对用户的影响：小模型触发主动定时器时更不容易把场景理解为被用户重复打扰。
  - 兼容性：仅优化系统提示词，不改变配置字段或 RPC 行为。

## 已废弃但仍兼容

- `runtime_http.py` 作为 HTTP / WebSocket 入口已不推荐直接使用。
  - 废弃对象：`python runtime_http.py ...`
  - 生效版本：`2026.6.21.0`。
  - 替代方案：使用 `python -m GensokyoAI.backends.web_server ...`。
  - 预计移除：暂未确定；移除前会另行通知。
  - Runtime 声明：未写入 `runtime.info.deprecated_methods`（此文件属于启动入口，非 Runtime 方法）。

- `tool.web_search.provider: "bing"` 不再是默认推荐，但仍兼容。
  - 废弃对象：`tool.web_search.provider` 默认值 `"bing"`。
  - 生效版本：`2026.6.21.0`。
  - 替代方案：默认 `"ddg"`；需要 API 接入时改用 `"api"` 或 `"mixed"`。
  - 预计移除：暂未确定。
  - Runtime 声明：未写入 `runtime.info.deprecated_fields`（仅默认值变化，字段本身未废弃）。

## 已知问题

- 当前版本仍处于 Alpha 阶段，Runtime API 已具备集成能力，但后续正式稳定版前仍可能做兼容性收口。
- `ddgs` 目前只提供同步 API，搜索调用在异步 Runtime 中通过 `asyncio.to_thread` 包装；极高并发场景下可能成为瓶颈，如需更高并发可考虑 `api` Provider 或自定义 Provider。
- 完整端到端 smoke test 和打包验收仍需在发布前继续执行并记录。
