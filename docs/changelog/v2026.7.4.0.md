# 版本 v2026.7.4.0

**发布日期**: 2026-07-04
**类型**: 功能更新
**建议操作**: 推荐升级。本次新增场景系统，完全向后兼容，旧配置、旧会话、旧角色卡均无需修改。

---

## 一句话总结

新增「场景系统」：把角色所处的环境（博丽神社、魔法森林等）外置为常驻的结构化状态，角色不再需要每轮对话都复述自己身处何地，从而降低角色扮演的上下文负担——小模型也能有更稳定的临场表现。同时把角色开场 `begin_scene` 升级为「场景 + 开场动作」，与场景系统打通。

---

## 新增功能

### 场景系统（SceneManager）

- **新增**：全局共享的场景库。在项目根目录 `scenes/` 下用 YAML 定义地点（名称、环境描述、氛围、时段、连通场景、关键物件），所有角色共用同一套幻想乡地点。
  - 对用户的影响：角色扮演时不必每句话强调"我在博丽神社"，环境作为常驻状态存在，角色的注意力可以更集中在人设与对话上。
  - 是否需要操作：默认关闭。需在配置中开启 `scene.enabled: true`，并在 `tool.builtin_tools` 中加入 `"scene"`，模型才能获得场景工具。

- **新增**：两个模型内置工具（均为异步、无状态，通过事件总线与 SceneManager 通信）。
  - `scene_switch(scene_id)`：剧情推进时角色主动切换所处场景。
  - `get_current_scene()`：角色"想不起来自己在哪"时主动查询当前场景。
  - 设计意图：场景只在"对话开始"注入一次（新会话首轮 / 恢复会话后首轮），之后不再每轮注入；角色遗忘时自行调用 `get_current_scene` 找回，几乎不增加持续的上下文成本。

- **新增**：当前场景随会话持久化。当前所处场景写入会话的 `metadata.current_scene_id`，退出重进（resume）后仍停留在上次的地点。

- **新增**：场景切换广播事件 `SystemEvent.SCENE_SWITCHED`，前端可订阅用于提示"当前场景：博丽神社"。内置 console 后端已接入，切换后会以灰色小字显示当前场景。

- **新增**：可选的连通性约束 `scene.enforce_connectivity`。开启后，角色只能切换到当前场景 `connected_scenes` 中列出的相邻场景（例如不能从博丽神社直接瞬移到迷途竹林）。

- **新增**：内置 5 个示例场景（`scenes/zh_cn/`）：博丽神社、魔法森林、红魔馆、迷途竹林、人间之里，带连通关系，开箱即用。

### begin_scene 升级为「场景 + 开场动作」

- **新增**：角色卡 `begin_scene` 支持结构化写法，与场景系统打通。
  ```yaml
  begin_scene:
    scene: hakurei_shrine        # 初始场景 id，交给场景系统设为起始场景并持久化
    action: "正在扫院子，红叶落了一地。"  # 角色此刻在做什么，驱动主动开场
  ```
  - 开场那一轮同时拿到「完整场景环境」（由场景系统自动注入）与「当前动作」，角色带着环境和状态自然开口。
  - 初始场景解析优先级：会话已保存的场景 > 角色 `begin_scene.scene` > `scene.default_scene` > 无。
  - 是否需要操作：仅在希望角色开场即置身某个场景时才需要填 `scene`。

## 行为变化

- **变化**：`begin_scene` 的类型从"纯字符串"扩展为"字符串或 `{scene, action}` 结构"。
  - 兼容性：**旧的纯字符串写法完全兼容**，会被自动归一化为"只有开场动作、不指定场景"，行为与升级前一致。无需修改任何现有角色卡。
  - 客户端影响：`character.validate` / `character.preview` 返回的预览结构中，`has_begin_scene` 语义不变（有内容为 true），并新增 `begin_scene_id` 字段（结构化写法下为场景 id，否则为 null）。
