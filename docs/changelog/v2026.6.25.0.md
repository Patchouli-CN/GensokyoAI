## 新增功能

### 角色卡 metadata 注入系统提示词

- **新增**：角色 YAML 中的 `metadata` 字段现在会被注入到系统提示词中，作为模型可用的额外上下文。
  - 对用户的影响：可以在角色卡中写入作者、版本、标签等元数据，模型在对话中可感知这些信息。
  - 是否需要操作：无需操作。已有角色卡如含 `metadata` 会自动生效；无 `metadata` 不影响功能。

### 角色卡 example_dialogue 作为 few-shot 注入

- **新增**：角色 YAML 中的 `example_dialogue` 字段现在会在每轮对话构建时作为 few-shot 示例注入到消息列表中（最多 3 条）。
  - 对用户的影响：模型能更好地模仿角色的对话风格，减少风格漂移。
  - 是否需要操作：无需操作。已有角色卡如含 `example_dialogue` 会自动生效。

### 框架级角色扮演视角约束

- **新增**：`_build_system_prompt()` 现在会在角色卡原有设定末尾追加一层框架规则，统一约束模型行为：
  1. **禁止第三人称描述自己** — 明确禁止「雾雨魔理沙微笑着说」等旁观者视角，要求以第一人称直接回应。
  2. **用户 `*` 星号动作是另一个角色的行为** — 告诉模型用户输入中的 `*微笑* 你好` 是另一个角色的动作，不要复述或模仿。
  3. **禁止重复用户内容** — 不要复述、概括、翻译用户已经说过的话。
  4. **禁止跳脱角色** — 不使用「作为AI」「根据设定」等旁观者语言。
  - 对用户的影响：AI 角色扮演的沉浸感显著提升，角色不会突然变成旁观者描述自己。
  - 是否需要操作：无需操作。所有角色自动获得此约束。

## 行为变化

### 主动定时器决策改为后台执行（不再阻塞输入）

- **变化**：`Agent._on_generate_response` 中的 `schedule_initiative_timer` 调用从 `await` 同步执行改为 `asyncio.create_task` 后台执行。
  - 对用户的影响：AI 回复完成后，输入行立即弹出，不再有约 5 秒的卡顿等待。
  - 兼容性：完全向后兼容。定时器决策仍在后台完成，不影响功能正确性。
  - 对客户端的影响：无。`complete_response` 仍按原有顺序触发。

### 流式响应轮询优化

- **变化**：`send_stream` 和 `send_multimodal_stream` 中的轮询逻辑从 `asyncio.wait_for(get_chunk, timeout=0.1)` 改为 `asyncio.wait([get_chunk, response_future], FIRST_COMPLETED)`。
  - 对用户的影响：`response_future` 完成后立即退出循环，不再硬等 0.1 秒超时。
  - 兼容性：完全向后兼容。超时和异常处理逻辑保持一致。

## 架构重构

### ThinkEngine 统一接管所有思考逻辑

- **重构前**：
  - ThinkEngine 只负责长期思考（每 5 分钟游走话题图谱）。
  - InitiativeTimer 直接调用 LLM 进行短期思考（回复后决策）和犹豫重试。
  - Agent 直接调用 LLM 进行说话前思考（定时器到期前的 thought prompt）。
- **重构后**：
  - **ThinkEngine** = 所有思考（长期 + 短期 + 说话前）。
  - **InitiativeTimerManager** = 纯定时器调度器（创建、取消、触发、管理），不再直接调用 LLM。
  - **Agent** = 协调层，将 ThinkEngine 注入 InitiativeTimerManager。
- 新增 ThinkEngine 方法：
  - `decide_initiative()` — 短期思考：AI 回复后，决策是否设置主动定时器（原 `InitiativeTimer._decide`）。
  - `pre_speak_thought()` — 说话前思考：定时器到期前，生成主动消息前的内部思考（原 `Agent._handle_initiative_timer_trigger` 中的 thought prompt）。
- 对开发者的影响：
  - `InitiativeTimerManager.__init__` 的 `model_client` 参数已移除，改为 `think_engine` 参数。
  - `ThinkEngine.__init__` 新增 `initiative_timer_config` 参数。
  - 无外部 API 或 Runtime 协议变化。

## 修复问题

- **修复** `json.dumps` 序列化不可序列化对象（如 method）导致的 `Object of type method is not JSON serializable` 崩溃。
  - 受影响场景：工作记忆中的工具调用消息包含不可序列化对象时，`logger.trace` 中的 `json.dumps` 会崩溃。
  - 升级建议：所有用户建议升级，此崩溃在对话进行工具调用后高概率触发。

- **修复** `except asyncio.CancelledError, GeneratorExit:` 的 Python 3 语法错误，改为 `except (asyncio.CancelledError, GeneratorExit):`。
  - 受影响场景：Python 3.14 严格模式下会报语法错误。

- **修复** `MessageBuilder` 中 `self._metadata = metadata or {}` 与 `@property` 之间缺少换行的缩进问题。

- **修复** ruff SIM105 警告：多处 `try-except-pass` 改为 `contextlib.suppress`。
