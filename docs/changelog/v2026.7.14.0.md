# GensokyoAI v2026.7.14.0 更新日志

发布日期：2026-07-14

## 发布范围

这是自此前唯一正式 release `v2026.5.13.0` 以来的累计正式发布。`v2026.6.21.0` 至 `v2026.6.25.1` 以及 `v2026.7.4.0` 文档仅是未发布开发快照或候选记录，不是独立正式 release；本记录汇总其已经进入当前代码基线的变化。

## 一句话总结

本次累计发布增加主动定时器、会话历史编辑与重生成、多模态消息、扩展角色资料、默认 DuckDuckGo 搜索和完整场景系统，同时完成 Runtime Web 入口迁移兼容、安全与性能收口、许可证调整及 Streamlit 净移除。

## 新增功能与行为变化

- **主动定时器**：AI 可在回复后安排稍后主动发言，支持积存摘要、更新、取消、立即触发、犹豫复判、自然兜底及 CLI/Runtime 状态管理。
- **会话历史编辑与重生成**：新增消息历史读取、整体替换及从指定位置重新生成能力，供前端安全实现编辑历史和重新回答。
- **多模态**：消息与 Provider 转换链路支持多模态内容，并在不同 Provider 的能力边界内保持兼容处理。
- **角色资料增强**：角色卡和角色包链路支持更丰富的 `metadata` 与 `example_dialogue`，校验和预览接口同步暴露摘要。
- **HTTP/WebSocket 迁移兼容入口**：Web 服务实现迁移到 `GensokyoAI.backends.web_server`；原有启动脚本和兼容入口继续可用，HTTP RPC、WebSocket 流式消息与事件订阅保持统一 Runtime 契约。
- **默认搜索切换**：内置 Web 搜索默认使用 DuckDuckGo/DDGS，不要求配置搜索服务密钥。
- **场景系统**：新增 YAML 场景库、`SceneManager`、会话当前场景持久化、场景切换事件、可选连通性约束、模型内置场景工具，以及结构化 `begin_scene`。
- **场景 RPC**：新增 `scene.current`、`scene.list`、`scene.get`、`scene.switch`、`scene.graph`，前端可查询、切换和可视化场景。
- **安全加固**：加强 Runtime Origin/token 部署校验、命令与路径策略、外部工具策略、提示注入边界、会话持久化路径和 SSRF/URL 检查；异步 URL 安全校验避免阻塞 HTTP Runtime。
- **性能优化**：复用 HTTP 连接、使用高性能序列化与异步文件 I/O、缓存角色配置、缩小锁粒度并改进并发状态管理。这里不列出未经本次发布流程重新测量的百分比。
- **许可证变化**：项目许可证调整为 Apache License 2.0；分发 metadata 与仓库许可证文件保持一致。
- **Streamlit 净移除**：曾在开发过程中加入的 Streamlit 前端及其残留已从最终累计发布中移除，不作为本版本能力或依赖交付。

## Runtime / 客户端兼容性

- Python package：`2026.7.14.0`；对应计划 tag：`v2026.7.14.0`。
- Runtime protocol：`1.1.0`；`protocol_major_version` 保持 `1`。
- 自 `v2026.5.13.0` 基线新增 **14 个**非 legacy RPC：
  - 会话：`session.messages`、`session.replace_messages`、`session.regenerate_from`（3 个）。
  - 主动定时器：`initiative_timer.current`、`initiative_timer.update`、`initiative_timer.cancel`、`initiative_timer.trigger`、`initiative_timer.hesitation`、`initiative_timer.hesitation.set`（6 个）。
  - 场景：`scene.current`、`scene.list`、`scene.get`、`scene.switch`、`scene.graph`（5 个）。
- 既有命名空间 RPC 与 legacy 兼容方法未移除；`runtime.info.breaking_changes` 仍为空，无 RPC breaking change。
- 客户端应先调用 `runtime.info`，以返回的 `methods`、`method_specs`、`capabilities` 和 schema versions 做能力协商。

## 数据迁移与 Schema

- Memory schema 从正式基线的 `1` 升至 `2`。读取旧 memory topic store 时会自动迁移到 schema 2，并在迁移前写入备份；迁移成功或失败会进入 `runtime.info.migration_diagnostics`。
- 若迁移失败，原异常会向调用方报告，诊断记录包含稳定错误信息；不要删除备份，应先保留原文件和 `.bak` 后根据诊断修复或回退。
- 其他公开 schema 保持 `1`：config、session、session export、character package。
- 升级前仍建议备份配置、角色、会话、记忆与自定义场景资源。

## 安装、依赖与部署提醒

- `ayafileio` 最低版本统一为 `>=1.4.6`。
- 核心依赖新增 `ddgs>=9.14.4`，用于默认 DuckDuckGo 搜索。
- Provider SDK 仍按 optional dependency 安装，最小依赖不强制安装全部 Provider。
- 对浏览器或跨来源客户端部署 Runtime 时，应显式配置允许的 Origin；不要用宽泛 Origin 放开不可信站点。
- 对非本机或不可信网络暴露 HTTP/WebSocket Runtime 时必须启用并妥善注入访问 token。不要把 token 写入仓库、日志、截图或前端公开资源。
- PyPI 上传本阶段暂缓；本次只准备代码、文档、测试及本地产物。

## 已移除或破坏性变化

- Streamlit 前端已净移除。
- Runtime RPC 无破坏性变化，protocol major 保持 `1`，breaking changes 保持空。
