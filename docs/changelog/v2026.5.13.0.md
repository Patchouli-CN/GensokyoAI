# GensokyoAI v2026.5.13.0 更新日志

发布日期：2026-05-13

## 一句话总结

这是 GensokyoAI 的首个公开 Alpha release。这个版本不是相对上一版的增量更新，而是确立当前可对外集成的初始能力基线：角色扮演 Agent、Provider 抽象、会话与记忆、工具调用、角色包、Runtime RPC、HTTP / WebSocket 入口、配置与角色诊断、迁移诊断和资源控制。

## 新增功能

- 首次发布 GensokyoAI Python 后端工具包与 Runtime API。
  - 对用户的影响：可以从源码或包入口运行角色扮演 Agent，也可以通过 Runtime RPC 集成到 CLI、桌面客户端、Web 服务或第三方程序。
  - 是否需要操作：需要 Python 3.14+；推荐使用 uv 管理环境。

- 提供多 Provider 模型接入能力。
  - 支持 Ollama、OpenAI、OpenRouter、DeepSeek、OpenAI Responses、Claude、Gemini。
  - Provider SDK 保持 optional dependencies，不会在最小安装中强制安装全部模型服务依赖。
  - 默认推荐本地 Ollama，可通过 `default` 或 `ollama` extra 安装。

- 提供角色扮演 Agent 核心能力。
  - 支持角色 YAML、人设提示词、问候语、示例对话、会话上下文、工具调用和流式回复。
  - 支持 `agent.init`、`agent.send_message`、`agent.send_message_stream` Runtime RPC。

- 提供会话、记忆和导出能力。
  - 支持创建、列出、恢复、当前会话查询、删除、导出、重命名和回滚会话。
  - 支持语义记忆列表、搜索、读取、更新、删除和话题图查询。
  - 会话导出包含机器可读 schema 信息，便于后续客户端和工具链处理。

- 提供配置、角色和角色包诊断。
  - `config.validate` 可在初始化 Agent 前返回结构化配置 diagnostics。
  - `character.validate` 可校验和预览角色 YAML。
  - `character.list` 可列出角色并保留坏文件诊断。
  - `.gensokyo-character` 角色包支持 validate、preview、import、export。

- 提供 Runtime 集成入口。
  - JSON Lines RPC：`bridge_main.py`。
  - HTTP / WebSocket adapter：`runtime_http.py`。
  - `GET /info` / `runtime.info` 暴露 package version、protocol version、methods、legacy methods、method specs、capabilities、schema versions、迁移诊断和资源控制摘要。
  - WebSocket `agent.send_message_stream` 可逐事件推送 Runtime stream 结果。

- 提供资源控制与结构化错误。
  - Runtime 入口和深层模型 / 工具调用具备资源 gate 与超时配置。
  - 资源限制触发时返回稳定 `resource.limit_exceeded` 结构化错误。
  - 工具错误、Runtime RPC 错误和方法不存在错误均提供稳定 error code、用户提示、恢复建议和 details。

## 行为变化

- 首版发布，无相对上一正式版本的行为变化。
- 当前版本处于 Alpha 阶段：核心能力已经可用于集成验证，但正式稳定版前仍可能继续收口文档、客户端兼容性和端到端验收。
- Runtime RPC 以命名空间方法作为推荐调用方式；旧的非命名空间方法仍兼容但已标记 deprecated。

## 修复问题

- 首版发布，无相对上一正式版本的修复项。
- 发布前收口中已修正文档中过于面向未来或不够准确的 Runtime 流式能力表述。
- 发布前收口中已将抽象基类占位实现从简单 `pass` 调整为明确 `raise NotImplementedError`，并补充抽象契约测试。

## 已废弃但仍兼容

- Legacy 非命名空间 Runtime RPC 方法已废弃但仍兼容。
  - 废弃对象：`init`、`send_message`、`send_message_stream`、`list_characters`、`create_session`、`list_sessions`、`current_session`、`resume_session`、`delete_session`、`export_session`、`rename_session`、`rollback_session`、`shutdown`、`dependency_status`、`install_dependencies`、`external_tool_status`。
  - 生效版本：`2026.5.13.0`。
  - 替代方案：使用 `runtime.info.method_specs[].replacement` 指向的命名空间方法，例如 `init` 改用 `agent.init`，`send_message` 改用 `agent.send_message`。
  - 预计移除：`2.0.0`。
  - Runtime 声明：已通过 `runtime.info.deprecated_methods` 和 `runtime.info.compatibility_notes` 暴露。

## 已移除或破坏性变化

- 无。
  - Runtime protocol major 保持 `1`。
  - `runtime.info.breaking_changes` 当前为空数组。

## 数据迁移与升级提醒

- 首版发布不包含从旧正式版本迁移到新正式版本的升级路径。
- 当前实现支持读取旧无版本会话或旧 memory topic store 时尽量迁移到当前 schema，并记录迁移诊断。
- 迁移诊断可通过 `runtime.info.migration_diagnostics` 查看。
- 建议升级或切换版本前备份 `config/local.yaml`、`characters/`、`sessions/`、记忆数据和自定义资源。

## 安装与依赖变化

- Python 版本要求：Python 3.14+。
- 推荐包管理器：uv。
- 最小运行依赖见 `requirements.txt` 和 `pyproject.toml` 的 `dependencies`。
- Provider SDK 为 optional dependencies：
  - `default` / `ollama`：Ollama。
  - `openai`、`openrouter`、`deepseek`、`openai_responses`：OpenAI SDK。
  - `claude`：Anthropic SDK。
  - `gemini`：Google GenAI SDK。
  - `all`：安装全部内置 Provider SDK。
- Windows 用户可使用 `run_default_uv.cmd` 或 `run_default_pip.cmd`。`run_default_pip.cmd` 使用前需要先安装对应 Provider extra，例如 `python -m pip install -e .[default]`。

  - `agent.init`、`agent.send_message`、`agent.send_message_stream`

  - `dependency.status`、`dependency.install`

  - `memory.list`、`memory.search`、`memory.get`、`memory.update`、`memory.delete`、`memory.graph`

- 废弃 methods / fields：legacy 非命名空间 RPC 方法已废弃；`deprecated_fields` 当前为空数组。
